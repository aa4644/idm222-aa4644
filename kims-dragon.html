Overview

This case study outlines the process and development of a mobile ordering web application created for Kim’s Dragon, a popular food truck located on Drexel University's campus. The project was developed as part of the IDM 216 (Interaction Design II) course, under the guidance of Professor Jervis Thompson. The goal of the project was to design and build a fully functional web application that would allow customers to place orders remotely, enhancing the food truck's operations and increasing customer satisfaction. As a team of five UXID students, we worked collaboratively, taking on different roles in project management, design, development, and data architecture to bring this web application from concept to completion.

Context & Challenge

Background

Kim’s Dragon, an Asian cuisine food truck on Drexel University's campus, is a favorite among students and local residents. It is located in “Food Truck Alley,” a busy street filled with competing food trucks. While the food truck is well-loved, it operates solely on an in-person, walk-up ordering system, which can be inefficient and lead to lost revenue, especially for customers in a hurry or those who want a quick and seamless ordering experience. Additionally, the lack of a digital ordering platform means that there is no way to streamline operations for the food truck staff.

The Problem

The absence of a mobile ordering system at Kim’s Dragon is the core issue this project sought to address. In a competitive environment with many food truck options nearby, it became essential for Kim’s Dragon to provide a way for customers to place orders digitally, without having to wait in long lines. This would not only cater to customers who are in a rush but would also improve the efficiency of the food truck’s operation by automating parts of the ordering process, reducing human error, and improving workflow.

Goals & Objectives

The main goal was to create a mobile web application for Kim’s Dragon that would offer an intuitive and seamless user experience. The project’s objectives included:

  • Developing a full-stack solution using HTML, CSS, JavaScript, and PHP
  • Creating an interactive design with an engaging user interface through usability testings
  • Ensuring that the final product could integrate seamlessly with Kim’s Dragon’s operational workflow, enabling efficient menu browsing, order management, and payment processing

Process & Insights: Beta to Final Build

Beginning the Project

We started the project by reviewing the materials collected in IDM 216, Interaction Design I, where the initial concepts for this project were created. Using that information, we adjusted our user persona and journey map to better align with the objectives for the project's development. We also created a style guide to experiment with the site's color palette, typography, and overall branding.

Alpha: Begining the Development

The Alpha phase involved refining the high-fidelity designs from IDM 215 (Interaction Design I), which had been created earlier and underwent one round of usability testing. During this phase, I developed a usability test script, conducted over five user interviews, and gathered valuable insights to inform the design. The team and I worked together to implement the initial front-end design using HTML and CSS. I focused specifically on creating components such as menus, buttons, forms, and navigation elements. This first iteration allowed us to get a feel for how the layout worked on mobile devices and desktops.


The Alpha build included fully stylized buttons, a basic menu system, and functional navigation between pages. As a secondary developer, I was responsible for building the microinteractions—things like smooth transitions for navigation, a functional counter button, and a loading bar. We used this early-stage prototype to test the core functionality and gauge user feedback.

View Alpha

Beta Description: Furthering the User Experience

Following the feedback received from the Alpha testing phase, we refined several aspects of the user interface. We also started integrating more dynamic features, including PHP for database integration. This allowed us to implement a cart system where users could add items from the menu, view item details, and calculate totals automatically based on selected items.


I contributed to developing features like the heart button (for saving favorite items), cleaning up CSS styles, and implementing a workable “Add Card Payment” feature. This feature allowed users to input and view their card information in real-time during checkout, which was a key step in moving the web application toward a fully functional e-commerce solution. With these changes in place, we moved to another round of usability testing, where we received positive feedback and gained further insights that helped us refine the product.

View Beta

Final: Finalizing the Product

The final version of the web application included a dynamic menu that displayed recipes with accompanying images and descriptions, all pulled from the database. Users could seamlessly browse the menu, add items to their cart, and complete the checkout process. The design was optimized for mobile devices and could be accessed easily from a smartphone or tablet. The final product was a fully responsive, mobile-friendly web app that supported secure transactions and provided a smooth, user-centered experience. The integration of PHP and JavaScript allowed for dynamic updates to the cart, real-time display of item totals, and a confirmation screen with order details upon successful checkout.

The Solution

The solution provided by this project was a fully functional mobile ordering web application designed to enhance the customer experience at Kim's Dragon. By allowing users to browse the menu, place orders, and make payments remotely, the web app alleviated the need for long in-person lines, improving the customer experience and increasing efficiency for the food truck staff. Additionally, the app's dynamic cart system and payment integration allowed Kim’s Dragon to streamline its operations, saving time and reducing human error.

The Results

The results of the project were a fully functional web application that met the needs of Kim's Dragon's customers and provided a seamless user experience for food truck staff.

  • Key Outcomes:
    • Seamless User Experience: The web application provided a smooth, intuitive experience for users, from browsing the menu to completing their order.
    • Positive Feedback: The usability testing consistently revealed that users appreciated the app's design, functionality, and ease of use. The feedback from both customers and food truck staff suggested that the app would be a valuable tool for the truck's operations.
    • Technological Integration: The use of PHP, JavaScript, and CSS allowed for a robust, full-stack solution that provided dynamic features like cart calculations and real-time payment processing.

Takeaways

This project was a comprehensive learning experience, providing insights into the entire design and development process. Working in a team environment with designated roles helped us collaborate effectively and produce a successful outcome. The iterative process of testing and refining our design based on user feedback was crucial to creating a product that met both user needs and technical requirements. From a personal perspective, my involvement as a secondary developer and tertiary designer allowed me to deepen my knowledge of front-end development, microinteractions, and usability testing, all of which were critical in delivering a polished final product. Most importantly, the project emphasized the importance of balancing user needs with technical limitations and business objectives, an essential skill for any interaction designer.
